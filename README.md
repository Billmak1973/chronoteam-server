# ⌚ ChronoTeam 專業二手名錶交易平台

## 📖 項目簡介
**ChronoTeam** 是一個致力於提供安全、透明、專業的二奢名錶交易生態圈平台。系統涵蓋了完整的買賣家交易閉環、豐富的社區互動機制、嚴格的帳號安全與處罰體系，以及高度定制化的管理員後台。
前端採用 **React (Hooks) + Thymeleaf** 混合渲染架構，後端基於 **Spring Boot + Spring Security + JPA**，並引入 **Redis** 進行緩存與分佈式會話管理，確保高併發下的數據一致性與系統安全性。

---

## 🚀 核心功能模塊

### 🛒 買家與商品中心
- **多維度商品瀏覽**：支持按分類、品牌、價格區間、成色進行篩選，支持多種排序（最新、價格升降序）。
- **商品詳情與變體切換**：支持同款不同成色（如近全新、極佳、良好等）一鍵切換，動態加載庫存與價格。
- **智能購物車 (React)**：
  - 導航欄下拉菜單與購物車詳情頁**雙向無刷新同步**。
  - 支持**樂觀更新**（數量調整、單選/全選結算、刪除），失敗自動回滾。
  - 庫存實時校驗與缺貨攔截提示。
- **訂單與支付閉環**：
  - 支持**線上模擬支付**與**線下門店支付**（中環/尖沙咀/銅鑼灣店）。
  - 結賬頁支持動態計算運費（後台可配置免郵門檻）、主/備用地址切換。
  - 訂單狀態追蹤（待付款/待線下付款/已支付），支持訂單明細展開、取消與刪除。
- **到貨通知系統**：缺貨商品一鍵訂閱，管理員補貨後一鍵批量發送系統通知。
- **瀏覽歷史與收藏**：自動記錄瀏覽軌跡（支持按時間段清理/批量刪除），一鍵收藏/取消收藏（帶心跳動畫）。

### 💰 賣家與財務中心
- **發佈出售申請**：支持多圖上傳（錶面、機芯、保卡、瑕疵特寫）、附件勾選、成色評估。
- **交易模式**：當前主打 **「平台直接買斷」** 模式（快速收款），代碼已預留寄售枚舉。
- **進度追蹤**：追蹤申請進度（待鑑定、已報價、已接受等），支持取消或徹底刪除記錄。
- **財務結算看板**：自動計算並展示累計總成交額、平台服務費扣除及實際淨收益（可提現狀態）。

### 💬 社區互動與評論系統 (React 深度定制)
- **富文本評論與回覆**：支持加粗、顏色、列表、連結等富文本編輯（自帶 XSS 防護），支持根評論與樓中樓回覆。
- **多層級互動**：支持點贊/踩、分頁、多種排序（最新、最早、最多點贊）。
- **防刷與速率限制 (Rate Limit)**：
  - 互動操作綁定速率限制，1分鐘內頻繁操作觸發**自動封禁**（如10分鐘）。
  - 封禁期滿後通過 `ScheduledExecutorService` 異步自動解封並歸檔歷史記錄。
- **評論管理與歸檔**：作者可修改/刪除評論；管理員刪除評論需填寫原因，系統自動發送通知並將原評論**歸檔審計**。
- **兩級舉報系統**：違法、色情、網暴、詐騙等分類選擇，防重複舉報攔截，舉報成功自動觸發系統通知紅點。

### 🛡️ 帳號安全與處罰體系
- **多維度安全設置**：
  - 支持原密碼或 **「安全問答」** 驗證修改密碼。
  - 帳戶安全等級評估（綁定郵箱/手機/問答綜合評分）。
  - **登錄日誌追蹤**：記錄真實 IP（支持代理頭解析）、設備類型、瀏覽器及 Session ID。
- **禁言與拉黑機制**：
  - **普通用戶**：雙向禁言（互相無法回覆，前端實時攔截）。
  - **管理員**：全局禁言（自定義時長：天/週/月）、永久拉黑（剝奪所有互動權限）。
- **申訴系統閉環**：用戶可針對禁言、拉黑、評論刪除提交申訴，管理員審核後自動解除或維持處罰，前端實時展示 6 種申訴狀態。
- **消息通知中心**：區分「系統通知」（處罰、刪除、到貨）與「互動消息」（回覆、@、點贊），導航欄實時紅點提示，一鍵標記已讀。

## 🛠️ 管理員後台功能

### ✅ 已實現
- **商品微調與推薦管理**：管理員可在詳情頁或後台直接修改商品資訊；支持設置**首頁精選熱賣權重**（智能排序）。
- **評論區執法**：一鍵置頂評論、刪除違規評論並選擇/填寫刪除原因（系統自動發送通知並歸檔）。
- **用戶處罰與申訴審核**：執行全局禁言、永久拉黑操作；查看並審核用戶的申訴請求（Tab 切換管理）。
- **到貨通知管理**：查看缺貨商品的等待名單，一鍵批量發送補貨系統通知。
- **系統動態配置**：實時調整基礎快遞費與免郵門檻，顧客結賬時立即生效。
- **站點外觀設置**：支持一鍵切換「白天（金色奢華）」與「晚上（科技霓虹/星空）」主題。
- **每日業務報表**：定時任務自動生成每日 GMV、訂單數、退款、新註冊用戶等統計數據。

## 💻 技術棧與架構亮點

### 前端技術
- **核心框架**：React 18 (Hooks, 狀態管理, 樂觀更新), Thymeleaf (SSR 模板引擎)
- **UI 與樣式**：HTML5/CSS3, Vanilla JS, FontAwesome 6, 自定義 CSS 動畫 (星空/霓虹/心跳)
- **富文本編輯**：`contentEditable` + `document.execCommand` (自研輕量級工具欄，解決 IME 輸入法 Bug)

### 後端技術
- **核心框架**：Spring Boot 3.x, Spring Security (自定義 `UserDetails` 與 Session 管理)
- **數據持久化**：Spring Data JPA, Hibernate, MySQL
- **緩存與分佈式**：Redis (Jackson JSON 序列化配置, 解決 `LocalDateTime` 與循環引用問題)
- **安全防護**：Jsoup (HTML 白名單過濾，徹底防禦 XSS 攻擊), BCrypt 密碼加密

### 架構亮點
1. **樂觀更新 (Optimistic UI)**：購物車、評論點贊等高频操作採用前端先渲染、後端異步校驗、失敗自動回滾的機制，極大提升用戶體驗。
2. **XSS 防禦機制**：評論區富文本提交時，後端使用 Jsoup 嚴格過濾 `style`、`class` 等屬性，確保前端渲染安全。
3. **異步定時任務**：使用 `ScheduledExecutorService` 處理速率限制的自動解封，避免阻塞主線程；使用 `@Scheduled` 處理每日報表與歷史數據清理。
4. **智能分頁算法**：後端封裝 `PageItem` 邏輯，前端實現 `1 ... 3, 4, 5 ... 10` 的省略號智能分頁 UI。
5. **軟刪除與審計**：評論刪除不直接物理丟失，而是寫入 `ReviewArchive` 歸檔表，保留操作人、原因及快照，便於後期審計。

## 📂 項目目錄結構 (簡述)
```text
├── src/main/java/org/example/website
│   ├── config/         # 安全配置、Redis配置、定時任務、數據初始化
│   ├── controller/     # RESTful API 與 Thymeleaf 頁面路由
│   ├── dto/            # 數據傳輸對象 (RegisterRequest, ApiResponse 等)
│   ├── entity/         # JPA 實體 (User, Product, Order, Review, AdminPenalty 等)
│   ├── repository/     # Spring Data JPA 接口與自定義 JPQL 查詢
│   ├── security/       # 自定義 UserDetails 實現
│   ├── service/        # 核心業務邏輯 (訂單、評論、處罰、速率限制等)
│   └── util/           # 工具類 (UID生成器、SecurityUtils)
├── src/main/resources
│   ├── static/css/     # 全局樣式、側邊欄、管理後台樣式
│   ├── static/js/      # 全局腳本 (common.js, admin.js, 評論表單組件)
│   ├── static/react-assets/ # React 編譯後的評論區組件
│   └── templates/      # Thymeleaf HTML 模板 (首頁、詳情、後台、個人中心等)
```

## 📜 開發與運行說明
1. **環境準備**：JDK 17+, MySQL 8.0+, Redis
2. **數據庫初始化**：啟動項目後，`DataInitializer` 會自動創建超級管理員帳號及基礎站點設置。
3. **前端構建**：React 組件（如評論區、購物車）通過 CDN 或本地構建引入，與 Thymeleaf 無縫融合。
4. **管理員登入**：默認帳號 `admin`（請在首次登入後修改）。
